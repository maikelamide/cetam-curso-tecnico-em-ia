## Instruções da atividade prática

Siga os passos abaixo:

1. Clonar o repositório para sua máquina.
2. Criar uma branch com o padrão: `dupla_nome1_nome2`.
3. Dentro da pasta `duplas`, criar um diretório com o mesmo nome da branch: `duplas/dupla_nome1_nome2`.
4. Dentro desse diretório, criar os arquivos:
   - `integrantes.txt`: contendo o nome dos dois integrantes da dupla.
   - `Questões.txt`: contendo as questões copiadas do arquivo `README.md`, com as respostas destacadas.
5. Realizar o versionamento, enviando a branch do repositório local para o remoto.
6. Abrir um PR (Pull Request) para a branch principal do projeto.
7. Criar uma nova branch com o padrão: `presenca_nome1_nome2`.
8. No diretório `chamada`, abrir o arquivo `frequência-da-prova.txt` e adicionar seu nome, sem excluir ou alterar o texto já existente.

## Questões da prova

### 1. O que é versionamento de arquivos?
a. Processo de apagar arquivos antigos para economizar espaço  
b. Controle de alterações em arquivos ao longo do tempo  
c. Criação automática de backups sem registro  
d. Organização de pastas no computador  

**Resposta:** b. Controle de alterações em arquivos ao longo do tempo

### 2. Sobre o Git, analise as afirmativas:

I. É um sistema de controle de versão que registra alterações em arquivos ao longo do tempo.  
II. Permite que várias pessoas trabalhem no mesmo projeto simultaneamente.  
III. Substitui a necessidade de salvar arquivos no computador.  

Assinale a alternativa correta:

a. Apenas I e II estão corretas  
b. Apenas I está correta  
c. Apenas II e III estão corretas  
d. I, II e III estão corretas  

**Resposta:** a. Apenas I e II estão corretas

### 3. Qual comando é utilizado para enviar alterações locais para o repositório remoto?
a. `git add`  
b. `git commit`  
c. `git push`  
d. `git clone`  

**Resposta:** c. `git push`

### 4. O que é o arquivo README em um projeto?
a. Um arquivo que executa o código automaticamente  
b. Um arquivo que armazena senhas do projeto  
c. Um arquivo usado apenas para backup  
d. Um arquivo de documentação com instruções e informações do projeto  

**Resposta:** d. Um arquivo de documentação com instruções e informações do projeto

### 5. O que é uma biblioteca em programação?
a. Um conjunto de códigos prontos que ajudam no desenvolvimento  
b. Um programa que instala o sistema operacional  
c. Um tipo de erro no sistema  
d. Um arquivo usado apenas para salvar textos  

**Resposta:** a. Um conjunto de códigos prontos que ajudam no desenvolvimento

### 6. Sobre repositório, analise as afirmativas:

I. É o local onde o projeto e seu histórico de versões são armazenados.  
II. Pode existir apenas na internet, não sendo possível localmente.  
III. Pode ser local (no computador) ou remoto (em plataformas como GitHub).  

Assinale a alternativa correta:

a. Apenas I está correta  
b. Apenas I e II estão corretas  
c. Apenas I e III estão corretas  
d. I, II e III estão corretas  

**Resposta:** c. Apenas I e III estão corretas

### 7. O que são dependências em um projeto?

a. Arquivos desnecessários que podem ser removidos  
b. Programas do sistema operacional  
c. Bibliotecas que o projeto precisa para funcionar  
d. Apenas arquivos de imagem  

**Resposta:** c. Bibliotecas que o projeto precisa para funcionar
